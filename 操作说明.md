# 操作说明

这份说明面向现场使用者：从编译、烧录、配网、绑定手柄到驾驶和 OTA 更新，按顺序执行即可。

## 1. 准备硬件

确认硬件连接完整：

- K10 已安装在 Maqueen Plus V2 上。
- Huskylens 已接到 K10 的 I2C。
- 手柄电量充足，并处于可广播状态。
- 小车放在安全位置，轮子悬空或远离桌边后再做首次测试。

## 2. 编译固件

在项目目录中运行：

```powershell
pio run
```

编译成功后会生成 `firmware.bin`。如果是第一次编译，PlatformIO 会自动下载 K10 平台和工具链。

## 3. 串口上传

先查看 K10 的串口号：

```powershell
pio device list
```

找到类似 `USB VID:PID=303A:1001` 的串口，例如 `COM26`，然后上传：

```powershell
pio run -t upload --upload-port COM26
```

上传成功后 K10 会自动复位并运行新固件。

## 4. 启动后的网络

固件启动后默认关闭 WiFi，只保留 BLE 手柄控制，以减少 2.4GHz 干扰和运行中误触 OTA 的风险。

需要配网、修改手柄绑定码或 OTA 时，同时长按 LT+RT 3 秒进入 OTA/配置模式。进入该模式后，小车会停车、断开手柄控制，并开启 AP 和 STA。

AP 固定信息：

- SSID: `Maqueen-OTA`
- 密码: `12345678`
- AP 地址: `192.168.4.1`

STA 默认信息：

- SSID: `DFRobot-guest`
- 密码: `dfrobot@2017`

如果默认 STA 可用，进入 OTA/配置模式后 K10 会自动连接。STA 地址可以从串口日志、路由器后台，或网页状态接口确认。

## 5. 配网和手柄绑定码

先在手柄上同时长按 LT+RT 3 秒。看到车辆停车并进入 OTA/配置模式后，电脑连接到 `Maqueen-OTA`，再用浏览器打开：

```text
http://192.168.4.1
```

页面中可以配置：

- STA SSID
- STA Password
- Pair Code

默认 Pair Code 是 `32652`。保存后配置会写入 K10 的持久存储，下次重启仍然生效。

修改 Pair Code 后，配置会保存到 K10。完成配置或 OTA 后，重启 K10 回到正常驾驶模式。

## 6. 绑定码规则

固件会扫描广播名为 `GamepadSpace-Q34B` 的 BLE 手柄，然后根据手柄 MAC 地址计算 5 位绑定码。只有计算结果等于当前 Pair Code 的手柄才会被连接。

现场换手柄时，只需要在网页里改 Pair Code，不需要重新编译固件。

## 7. 驾驶流程

上电后车辆默认不响应摇杆驱动，需要先进入可驾驶状态。

操作顺序：

1. 打开手柄，等待 K10 自动连接。
2. 按 LT，Huskylens 开始录像，小车进入可驾驶状态。
3. 使用左摇杆 Y 控制前进/后退。
4. 使用右摇杆 X 控制转向。
5. 使用右摇杆 Y 切换速度上限：
   - 推上：普通速度。
   - 拉下：最高速度。
6. 按 RT，Huskylens 停止录像，小车停车并退出可驾驶状态。

蓝牙断开、进入 OTA/配置模式、OTA 上传中或按 RT 停止时，固件都会执行停车保护。

同时长按 LT+RT 3 秒会进入 OTA/配置模式。这个操作会停止录像状态、停车、关闭驾驶控制，然后开启 WiFi。

## 8. 运动控制策略

当前固件使用曲率转向策略：

- 前进或后退时，通过降低内侧轮速度来转向。
- 原地转向时，两侧轮反向转动。
- 高速转弯会限制转向幅度，避免突然反向导致抖动。
- 全速原地转向会限制最高输出，降低跳动风险。
- 轮速突变、左右轮速差和加速度强度突变会触发牵引限制，之后逐步恢复输出。

驾驶时，固件会根据左右轮速差驱动手柄震动。左轮快于右轮表示右转，手柄右马达震动；右轮快于左轮表示左转，手柄左马达震动；轮速差越大，震动越强。两边轮速接近、停车或退出驾驶状态时会停止震动。

## 9. OTA 更新

OTA 服务默认关闭。需要上传前，先在手柄上同时长按 LT+RT 3 秒进入 OTA/配置模式。

### AP 模式 OTA

进入 OTA/配置模式后，电脑连接 `Maqueen-OTA`，然后运行：

```powershell
curl.exe --noproxy "*" -f -S -m 180 -F "file=@.pio/build/unihiker_k10/firmware.bin" "http://192.168.4.1/ota?token=80794"
```

### STA 模式 OTA

进入 OTA/配置模式后，如果 K10 成功连接 STA，电脑和 K10 在同一网络内时，可以使用 K10 的 STA 地址：

```powershell
curl.exe --noproxy "*" -f -S -m 180 -F "file=@.pio/build/unihiker_k10/firmware.bin" "http://<K10_STA_IP>/ota?token=80794"
```

例如：

```powershell
curl.exe --noproxy "*" -f -S -m 180 -F "file=@.pio/build/unihiker_k10/firmware.bin" "http://192.168.9.37/ota?token=80794"
```

上传成功返回 `OK`，K10 会自动重启。重启后 WiFi 会再次关闭，回到 BLE 手柄控制模式。

## 10. 状态检查

访问状态接口：

```text
http://192.168.4.1/status
```

或：

```text
http://<K10_STA_IP>/status
```

返回内容包含：

- AP 地址
- STA 地址
- 是否正在上传 OTA

串口监视器也会输出手柄扫描、连接、OTA 上传和重启状态。

## 11. 常见问题

### 看不到串口

重新插拔 K10，确认数据线支持数据传输，然后运行：

```powershell
pio device list
```

### OTA 地址连不上

先确认电脑和 K10 在同一网络内。若 STA 地址不可达，改用 AP 模式：电脑连接 `Maqueen-OTA` 后访问 `192.168.4.1`。

### 手柄连不上

检查三点：

- 手柄广播名是否为 `GamepadSpace-Q34B`。
- 网页里的 Pair Code 是否正确。
- 手柄是否已经被其他设备连接。

修改 Pair Code 后可以重启 K10，或等待固件重新扫描。

### 车辆不动

确认已经按 LT 进入可驾驶状态。上电后默认是停止状态，摇杆不会直接驱动车辆。

### 高速转向打滑

当前固件已经做牵引限制。若仍然明显打滑，可以先降低最高速度测试，再根据轮胎、地面和负载继续调参数。

## 12. 安全注意

- OTA 或调试时让车轮离地，避免突然运动。
- 高速模式只在地面空间足够时使用。
- 蓝牙断连会停车，但惯性仍然存在，测试时不要让车辆靠近桌边。
- 不要在 OTA 上传过程中断电。
